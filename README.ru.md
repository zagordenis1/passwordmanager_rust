# `passwordmanagerrs` — Rust-порт [`passwordmanagerpy`](https://github.com/zagordenis/passwordmanagerpy)

**Языки:** [Українська](README.md) · **Русский** · [English](README.en.md)

[![CI](https://github.com/zagordenis1/passwordmanager_rust/actions/workflows/ci.yml/badge.svg)](https://github.com/zagordenis1/passwordmanager_rust/actions/workflows/ci.yml)

Небольшой CLI-менеджер паролей, написанный на идиоматичном Rust.
**Побайтово совместим** с оригинальной Python-реализацией: любую БД,
созданную одной из сторон, можно открыть другой (при совпадении
master-пароля). Базовый KDF — Argon2id; PBKDF2-SHA256 сохранён как
read-only fallback для старых БД.

> **Внимание про язык интерфейса.** Сам интерактивный CLI отображает
> сообщения **на украинском** (это сделано для побайтовой совместимости
> с оригинальной Python-программой и не меняется в Rust-порте). Этот
> README объясняет, что эти сообщения значат и как пользоваться
> утилитой по-русски.

> 📖 **Полная инструкция с пошаговыми сценариями:** [`USAGE.ru.md`](USAGE.ru.md)
> — каждая подкоманда, каждый пункт меню, edge-cases экспорта/импорта,
> бэкап, скрипты, troubleshooting.

## Установка Rust

Если Rust ещё не установлен — самый простой путь через [rustup](https://rustup.rs/).

### Linux / macOS

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
# Перезапустите shell или выполните:
source "$HOME/.cargo/env"
rustc --version   # должно быть 1.74 или новее
```

### Windows

1. Скачайте `rustup-init.exe` со страницы https://rustup.rs/.
2. Запустите, выберите `default host triple` (обычно `x86_64-pc-windows-msvc`).
3. Установите [Build Tools для Visual Studio](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (Microsoft C++ Build Tools) — нужны для линкера.
4. Проверьте в новой PowerShell-сессии: `rustc --version`.

### Системные зависимости

Никаких — Rust-сборка статически линкует SQLite (`rusqlite/bundled`) и
OpenSSL (`vendored`), поэтому отдельные пакеты `libssl` или
`libsqlite3` НЕ нужны. На Linux достаточно стандартного toolchain
(`gcc`, `make`), который обычно уже установлен.

## Сборка и запуск

### Клонирование

```bash
git clone https://github.com/zagordenis1/passwordmanager_rust.git
cd passwordmanager_rust
```

### Дебаг-сборка (для разработки)

```bash
cargo build
./target/debug/pwm --help
```

### Релизная сборка (быстрая, ~7-10 МиБ)

```bash
cargo build --release
./target/release/pwm --help
```

На Windows бинарник будет в `target\release\pwm.exe`.

### Установка в `~/.cargo/bin` (чтобы запускать как `pwm`)

```bash
cargo install --path .
pwm --help
```

> **Проверьте, что `~/.cargo/bin` есть в `$PATH`.** На macOS / Linux
> Bash или Zsh добавит это автоматически после установки rustup; если
> нет — добавьте `export PATH="$HOME/.cargo/bin:$PATH"` в `~/.bashrc` /
> `~/.zshrc`.

### Первый запуск

```bash
# Создаёт users.db в текущей директории и спрашивает master-пароль
pwm

# Или сразу через подкоманду:
pwm init
```

Путь к БД можно переопределить:

```bash
pwm --db /home/user/.local/share/pwm/users.db
pwm --db ./test.db init
```

## Графический интерфейс (`pwm-gui`)

Помимо CLI-бинарника `pwm` репозиторий содержит **необязательный**
графический интерфейс `pwm-gui` — desktop-приложение на egui/eframe.
Использует ту же библиотеку (`passwordmanagerrs`) и тот же формат БД,
поэтому CLI и GUI взаимозаменяемы: то, что добавлено в GUI, читается
из `pwm list`, и наоборот.

### Системные зависимости GUI на Linux

В дополнение к зависимостям CLI:

```bash
sudo apt install libgtk-3-dev libxkbcommon-dev libxcb-render0-dev \
                 libxcb-shape0-dev libxcb-xfixes0-dev libwayland-dev libssl-dev
```

На macOS и Windows дополнительные пакеты не нужны.

### Сборка и запуск

GUI отключён по умолчанию — обычная `cargo build` остаётся быстрой и
не тянет графические зависимости. Включите фичу `gui` явно:

```bash
cargo build --release --features gui --bin pwm-gui
./target/release/pwm-gui                                # ./users.db
./target/release/pwm-gui /home/user/.local/share/pwm/users.db
```

Установка в `~/.cargo/bin`:

```bash
cargo install --path . --features gui --bin pwm-gui
pwm-gui
```

### Возможности

- Создание мастер-пароля при первом запуске, разблокировка на
  последующих (с подтверждением и сообщением об ошибке).
- Список аккаунтов слева, детали справа: логин / email / пароль
  (скрытый по умолчанию; кнопки **Показать** и **Копировать**).
- Добавление аккаунтов через модальное окно со встроенным генератором
  (длина 4–4096, классы символов, пользовательский набор).
- Обновление пароля и удаление аккаунта с подтверждением.
- Отдельный генератор в меню (без добавления аккаунта).
- Смена мастер-пароля с re-encrypt всех строк в одной транзакции.
- Экспорт в JSON и импорт из JSON через нативный file picker
  (на Unix экспортный файл создаётся с режимом `0600`).
- Быстрый поиск по логину и email.
- Кнопка **🔒 Заблокировать** + auto-lock после
  `PM_AUTO_LOCK_SECONDS` секунд бездействия UI (по умолчанию 300 с;
  `0` отключает).
- Копирование в системный буфер обмена с автоматической очисткой
  через 30 секунд (отсчёт показан в статус-баре).

### Совместимость с CLI

База одна и та же: `users.db`. Можно, например, создать мастер-пароль
в GUI, а затем автоматизировать импорт через `pwm import` в скрипте
— и наоборот. Auto-lock в GUI считается от UI-активности
(мышь / клавиши); в CLI — от интерактивного ввода. Обе версии
учитывают переменную окружения `PM_AUTO_LOCK_SECONDS`.

### Размер релизного бинарника

`pwm-gui` весит ~15–17 МиБ (против ~7 МиБ у `pwm`) — это цена
встроенных шрифтов, GPU-бэкенда и нативных диалогов. Если
графический интерфейс не нужен — просто не включайте фичу `gui`,
бинарник `pwm` от этого никак не меняется.

## Интерактивное меню

Запуск без подкоманды открывает 11-пунктовое меню (точная копия
Python-версии, на украинском):

```
=== Password Manager ===
1)  Додати акаунт          — Добавить аккаунт
2)  Знайти акаунт          — Найти аккаунт по login
3)  Показати всі акаунти   — Показать все аккаунты
4)  Видалити акаунт        — Удалить аккаунт
5)  Оновити пароль         — Обновить пароль
6)  Експорт у JSON         — Экспорт в JSON
7)  Імпорт з JSON          — Импорт из JSON
8)  Пошук по login/email   — Поиск по login/email
9)  Вихід                  — Выход
10) Змінити master password — Сменить master-пароль
11) Згенерувати пароль     — Сгенерировать пароль
```

Master-пароль вводится один раз на сессию. После
`PM_AUTO_LOCK_SECONDS` секунд бездействия сессия блокируется и нужно
аутентифицироваться повторно.

### Подсказки по украинским промптам

| Что видите                        | Перевод                                      |
|-----------------------------------|----------------------------------------------|
| `Виберіть пункт:`                 | Выберите пункт                               |
| `Master password:`                | Master-пароль                                |
| `Новий master password:`          | Новый master-пароль                          |
| `Підтвердіть master password:`    | Подтвердите master-пароль                    |
| `Login:` / `Email:` / `Password:` | (как в английском)                           |
| `Не знайдено.`                    | Не найдено                                   |
| `Створено:` / `Оновлено.`         | Создано / Обновлено                          |
| `Видалено.`                       | Удалено                                      |
| `Невірний пароль.`                | Неверный пароль                              |
| `Перервано.`                      | Прервано (Ctrl+C / Ctrl+D)                   |
| `До побачення.`                   | До свидания                                  |
| `Сесію заблоковано через бездіяльність` | Сессия заблокирована из-за бездействия |

## Не-интерактивные подкоманды

Удобно для скриптов и pipe-композиции:

| Команда                      | Что делает                                                          |
|------------------------------|---------------------------------------------------------------------|
| *(без подкоманды)*           | Интерактивное меню (11 пунктов, на украинском).                     |
| `pwm init`                   | Первичная настройка master-пароля на свежей БД.                     |
| `pwm add --login <name>`     | Добавить аккаунт. Пароль через скрытый prompt или `--stdin`.        |
| `pwm get <login>`            | Вывести пароль в stdout (`--full` — полная запись).                 |
| `pwm list [--json]`          | Все аккаунты (с опцией JSON-вывода).                                |
| `pwm rm <login> [--force]`   | Удалить аккаунт. `--force` пропускает y/N-подтверждение.            |
| `pwm update <login>`         | Заменить пароль для существующего login.                            |
| `pwm change-master`          | Сменить master-пароль (атомарно перешифровывает все строки).        |
| `pwm gen [--length N]`       | Сгенерировать пароль и вывести в stdout.                            |
| `pwm search <query>`         | Подстроковый поиск по login + email.                                |
| `pwm export <path>`          | Экспорт в JSON (UNIX: файл создаётся с правами `0600`).              |
| `pwm import <path>`          | Импорт из JSON (дубликаты пропускаются по умолчанию).               |

### Примеры

```bash
# Добавить аккаунт, пароль читается со stdin (удобно для CI):
echo "supersecret" | pwm add --login alice --email alice@example.com --stdin

# Скопировать пароль в буфер обмена (Linux, Wayland):
pwm get alice | wl-copy

# Скопировать пароль в буфер обмена (macOS):
pwm get alice | pbcopy

# Сгенерировать 32-символьный пароль со всеми классами:
pwm gen --length 32

# Сгенерировать только буквы + цифры (без символов):
pwm gen --length 24 --no-symbols

# Экспорт + импорт между двумя БД:
pwm --db a.db export backup.json
pwm --db b.db import backup.json

# JSON-вывод для обработки jq:
pwm list --json | jq '.[] | .login'
```

### Флаги

* `--db <путь>` — глобальный флаг для любой подкоманды (по умолчанию `users.db` в текущей директории).
* `--version` / `-V` — версия бинарника.
* `--help` / `-h` — справка по любой подкоманде (например `pwm add --help`).

Master-пароль всегда читается через
[`rpassword`](https://docs.rs/rpassword) в TTY-режиме (без эха,
не сохраняется в истории shell). В не-TTY режиме (pipe) — из одной
строки stdin, поэтому `echo "pw" | pwm get foo` работает.

## Auto-lock (автоматическая блокировка)

Интерактивное меню имеет таймер бездействия, настраивается через
`PM_AUTO_LOCK_SECONDS`:

| Значение      | Что значит                                                  |
|---------------|-------------------------------------------------------------|
| не задано     | По умолчанию 300 секунд (5 минут).                          |
| `0`           | Полностью отключить auto-lock.                              |
| положительное | Блокировать после N секунд бездействия.                     |
| некорректное  | Откатывается к значению по умолчанию, никогда не отключает. |

Пункты 9 (выход) и 11 (генератор) помечены как `NO_AUTH_ACTIONS` —
они НЕ сбрасывают таймер бездействия и не требуют повторной
аутентификации.

```bash
# Непрерывная сессия без блокировки:
PM_AUTO_LOCK_SECONDS=0 pwm

# Блокировка через 60 секунд:
PM_AUTO_LOCK_SECONDS=60 pwm
```

> **Известное ограничение.** Таймер проверяется только между действиями
> пользователя. Если вы оставили бинарник на промпте меню — ключ
> остаётся в RAM до следующего нажатия клавиши. Подробности в
> `docs/audit-2026-04.md` (M3).

## Модель безопасности

| Аспект                              | Как решается                                                                                                       |
|-------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| Master-пароль в памяти              | Передаётся как `&str` только на время необходимости; на структуре менеджера не хранится.                            |
| Производный ключ KDF (base64-string)| Локальная копия `zeroize()`-ится сразу после создания Fernet.                                                       |
| Активный ключ шифрования            | Хранится в `Option<Fernet>`. Upstream-крейт `fernet` НЕ zeroize-ит — поэтому сбрасываем через `lock()`.             |
| Базовый KDF                         | **Argon2id, m=19456 KiB, t=2, p=1, hash_len=32, salt=16 байт** (рекомендация OWASP 2024).                            |
| Legacy KDF (только чтение)          | PBKDF2-HMAC-SHA256, 480 000 итераций.                                                                              |
| Шифрование                          | Fernet (AES-128-CBC + HMAC-SHA256), побайтово совместимо с Python `cryptography.fernet`.                            |
| Verifier (проверка пароля)          | Шифрованный известный plaintext `b"password_manager:verifier:v1"`.                                                  |
| Атомарность master-setup            | `BEGIN IMMEDIATE` + recheck внутри транзакции — защита от параллельных `pwm init`.                                  |
| Права на экспортный файл            | `0600` (только владелец) на Unix; выводится предупреждение о plaintext-содержимом.                                  |
| Durability SQLite                   | `journal_mode=WAL`, `synchronous=FULL`, `foreign_keys=ON`.                                                          |
| Auto-lock                           | Таймер бездействия в интерактивном меню (см. выше).                                                                  |
| Логирование                         | `Debug` redact-ит секреты; master / ключ / plaintext не логируются нигде.                                            |
| `unsafe`                            | Ни одного.                                                                                                          |
| `unwrap` / `expect` в проде         | Ни одного — только в тестах.                                                                                        |

### Threat model

**В scope:**

* Украденная `users.db` без master-пароля → атакующему придётся ломать Argon2id с OWASP-параметрами.
* Случайные swap-файлы / coredump → master и базовый ключ обнуляются (насколько позволяет upstream `fernet`).
* Конкурентный запуск `pwm init` → транзакция с reserved-локом не даст испортить БД.
* Экспортный JSON в общей директории на Unix → mode `0600` блокирует посторонних читателей.

**Вне scope** (защищайтесь средствами ОС):

* Дамп памяти привилегированным атакующим на запущенном процессе.
* Скомпрометированный компилятор / supply chain (`cargo audit` помогает, но не полностью).
* Кейлоггеры / shoulder surfing.

Полный аудит — `docs/audit-2026-04.md`.

## Архитектура

```
src/
├── lib.rs           # Library re-exports (бинарник + тесты)
├── main.rs          # Точка входа `pwm`
├── crypto.rs        # Argon2id + PBKDF2 + Fernet helpers + verifier
├── db.rs            # SQLite-схема и meta-аксессоры
├── manager.rs       # `PasswordManager`: lock/unlock/CRUD lifecycle
├── generator.rs     # Генератор паролей на CSPRNG
└── cli/
    ├── mod.rs          # Парсер `clap` + диспетчер
    ├── interactive.rs  # 11-пунктовое украинское меню
    └── commands.rs     # Не-интерактивные подкоманды

tests/
├── manager_tests.rs       # Интеграционные тесты публичного API
├── e2e_cli.rs             # Реальный бинарник через `assert_cmd`
├── auto_lock.rs           # Семантика auto-lock
└── cross_compat_python.rs # Открываем Python-БД и наоборот
```

## Тесты

```bash
cargo test                       # Все тесты
cargo test --lib                 # Только unit-тесты
cargo test --test e2e_cli        # Только CLI e2e
cargo clippy --all-targets -- -D warnings
cargo fmt --all -- --check
```

`cross_compat_python` пропускается, если `python3` отсутствует в `$PATH`
или если `cryptography` / `argon2-cffi` нельзя импортировать. В CI они
устанавливаются, поэтому suite выполняется безусловно.

Всего 56 тестов: 38 unit + 3 интеграционных + 10 e2e + 2 auto-lock + 3
cross-compat.

## Релизный бинарник

```bash
cargo build --release
ls -lh target/release/pwm        # Linux / macOS
dir target\release\pwm.exe       # Windows (PowerShell)
```

Профиль `release` в `Cargo.toml` применяет `lto = "thin"`,
`codegen-units = 1`, `strip = true` — итоговый статический бинарник на
x86_64 Linux меньше 10 МиБ (обычно 7-8 МиБ).

## Миграция с Python-версии

Смотрите [`MIGRATION.md`](MIGRATION.md). Коротко: скопируйте свою
`users.db` туда, где запускается Rust-бинарник — ничего
перешифровывать не нужно.

```bash
cp ~/old-python-pwm/users.db ./users.db
pwm list   # появится предупреждение про PBKDF2 → запустите `pwm change-master` для миграции на Argon2id
```
